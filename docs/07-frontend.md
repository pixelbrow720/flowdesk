# 07 — Frontend (`@flowdesk/web`)

Next.js 14 app in `apps/web/`. Renders the terminal: a WebGL exposure **heatmap**,
per-strike **profiles**, **levels**, and the **auth** flow. Consumes Snapshots
(validated against `@flowdesk/contracts`).

## Layout

```
app/         Next.js routes (App Router), layout, auth pages, the terminal view
components/  heatmap canvas, exposure profiles, level markers, HUD, auth UI
lib/         data loading (session JSON / WS), snapshot validation, helpers
public/sessions/  prebuilt session Snapshot JSON (ES/NQ_2026-06-0{1,9}.json)
```

## Design system

All visuals come from `@flowdesk/tokens` (locked):

- Turquoise `#40E0D0` (positive/call/long-gamma), Crimson `#E0183C`
  (negative/put/short-gamma), base `#000000`.
- Space Grotesk (UI) + JetBrains Mono (numbers). **Never Inter.**
- Tailwind preset from the tokens package enforces this in code.

## Rendering

- The **heatmap** is WebGL (custom shaders) driven by `field.price_grid / gamma /
  delta`. The Snapshot invariant `len(price_grid)==len(gamma)==len(delta)` is
  what makes the buffers safe to upload directly.
- **Profiles** render `profile[]` (`net_gex` / `net_dex` per strike).
- **Levels** overlay call/put walls, gamma flip, largest GEX/DEX from `levels`.
- `state` / `stale` / `expired` drive the dim/freeze states of the UI.

## Data sources

- **Dev / demo:** static session JSON in `public/sessions/` (generated by
  `scripts/gen_session_snapshots.py`). Lets the FE run with no backend.
- **Live path:** WebSocket subscription to the API (per instrument), with REST
  for the initial latest/historical snapshot.

## What's built vs. remaining

**Built:** the design-token system, the WebGL heatmap primitive + shaders, the
exposure-profile components, level markers, the auth UI, and session-JSON
loading. Component/unit tests run via `pnpm --filter @flowdesk/web test`.

**Remaining (largest FE work):** assembling the full **TRACE-style dashboard**
to match the `1.png` reference — the integrated layout, the intraday **HIRO line**
render, the timeline scrubber polish, and live-WS wiring end-to-end. See
[`08-status-and-gaps.md`](08-status-and-gaps.md) and [`09-roadmap.md`](09-roadmap.md).
